# Normas del equipo (MVP Asistente IA)

## Convenciones
- Python: `snake_case` en funciones/variables.
- Un módulo por responsabilidad (`db`, `tools`, `intents`, `orchestrator`, `app`).
- Commits cortos con prefijos sugeridos: `feat:`, `fix:`, `docs:`, `test:`.

## Definición de "hecho"
Una tarea está completa si:
1. Tiene código + prueba mínima (si aplica).
2. No rompe ejecución local.
3. Está documentada en README o en el issue.

## Revisión semanal
- Demo interna cada viernes.
- Se priorizan bugs críticos antes de nuevas features.
